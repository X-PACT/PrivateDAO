# Pricing Model

This is the current PrivateDAO pricing model hypothesis for pilots and early production.

## 1. Pilot package

For the earliest organizations, pricing should be tied to:

- onboarding
- configuration
- operator support
- evidence and review packaging

## 2. Hosted operator layer

For organizations that want a managed operating surface:

- hosted read-node path
- runtime reporting
- review and export tooling
- proposal and payout workflow support

## 3. Confidential operations premium

Additional pricing can attach to:

- confidential payroll workflows
- confidential bonus workflows
- REFHE-bound execution workflows
- private token corridor workflows

## 4. Enterprise or governance retainer

For serious organizations:

- governance configuration
- operator support
- quarterly review support
- security and launch coordination

## Honest boundary

PrivateDAO does not yet claim a live commercial pricing page or signed customer contracts in the repository.

This document is the product and revenue model for pilots, not a claim of existing paid production revenue.
