# Service Catalog

PrivateDAO can be packaged as product software, operator tooling, and launch support. This page is a structured catalog of what can be offered based on the current repository state.

## 1. Product surfaces

These are already implemented in the repository and live on Devnet:

- wallet-connected governance frontend
- DAO bootstrap and treasury funding
- proposal creation, commit, reveal, finalize, and execute
- confidential payroll and bonus proposal flows
- ZK companion review path
- REFHE envelope binding
- MagicBlock corridor binding for confidential token payouts
- backend-indexed read path with direct RPC fallback

## 2. Hosted API and ops surfaces

These are service-style offerings that build on the implemented product:

- hosted read-node and proposal API
- reviewer export and evidence packaging
- runtime diagnostics and operator snapshots
- governance proof-center support for audits, grants, and internal reviews

Current boundary:

- the repository documents and demonstrates these surfaces
- this is not presented as a public self-serve SaaS checkout yet

## 3. Migration and onboarding services

PrivateDAO can also be delivered as a guided migration and rollout package:

- Realms-oriented migration planning
- governance token mapping and custody planning
- operator onboarding
- launch blocker tracking
- pilot-run simulation on Devnet

Supporting docs:

- [Migration Story](migration-story.md)
- [Pilot Program](pilot-program.md)
- [Pilot Onboarding Playbook](pilot-onboarding-playbook.md)

## 4. Confidential operations premium

The strongest premium operating layer is the confidential treasury path:

- encrypted payroll approvals
- encrypted bonus approvals
- proposal-bound REFHE evaluation inputs
- proposal-bound MagicBlock settlement gating
- audit-oriented trust packaging

These are best positioned for:

- enterprise-style DAOs
- crypto-native funds
- grant committees with sensitive allocations
- gaming and token distribution workflows

## 5. Commercial packaging

The current commercial model is organized as:

- pilot package
- hosted operator layer
- confidential operations premium
- enterprise governance retainer

See:

- [Pricing Model](pricing-model.md)

## Honest boundary

This document does not claim:

- live paying customers in the repository
- a public billing portal
- production mainnet custody completion

It documents the services and packaging that can be truthfully offered from the current product and operational surface.
